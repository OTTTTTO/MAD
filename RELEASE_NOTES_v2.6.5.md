# MAD v2.6.5 发布说明

## 🎉 讨论列表排序功能

**发布日期：** 2025-02-02

### ✨ 新功能

#### 📊 讨论列表排序
为讨论列表添加了灵活的排序功能，帮助用户更好地组织和管理讨论。

**支持的排序方式：**
- **最新创建** - 按创建时间降序排列（默认）
- **最早创建** - 按创建时间升序排列
- **消息数（多→少）** - 按消息数量降序排列
- **消息数（少→多）** - 按消息数量升序排列
- **持续时间（长→短）** - 按讨论持续时间降序排列
- **持续时间（短→长）** - 按讨论持续时间升序排列

**功能特点：**
- ✅ 实时排序 - 选择排序方式后立即应用
- ✅ 与搜索联动 - 排序功能同样适用于搜索结果
- ✅ UI 优化 - 美观的下拉选择器，自适应主题
- ✅ 性能优化 - 前端快速排序，无需服务器额外请求

### 🎨 UI/UX 改进
- 在搜索框下方添加了排序选择器
- 优化了排序控件的视觉样式
- 提升了整体用户操作体验

### 🔧 技术实现
- 新增 `sortDiscussions()` 函数处理排序逻辑
- 修改 `loadDiscussions()` 函数支持动态排序
- 修改 `performSearch()` 函数使搜索结果也支持排序
- 添加排序选择器的事件监听
- 新增 CSS 样式 `.sort-options` 和 `.sort-select`

### 📝 使用方法
1. 在左侧边栏的搜索框下方找到排序选择器
2. 点击下拉菜单选择排序方式
3. 列表会立即按照选定的方式重新排序
4. 搜索时也会保持当前选择的排序方式

- 排序偏好不会被保存（每次刷新页面会重置为默认）
- 排序功能完全在前端实现，不影响后端性能
